# 15 - Guía para Contribuir

## Code Style

- Usar `rustfmt` antes de commit
- Seguir convenciones Rust estándar
- Documentar funciones públicas

## Branch Strategy

- `main` - Producción
- `develop` - Development
- `feature/*` - Nuevas features
- `fix/*` - Bug fixes

## Pull Request

1. Fork el repo
2. Crear branch feature
3. Hacer cambios
4. Agregar tests
5. Abrir PR a `develop`
